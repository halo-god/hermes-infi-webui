## 三、我补充的发布前必须确认事项

### 3.1 技术配置（**高优先级**）

| 项目 | 当前状态 | 必须确认 | 风险等级 |
|------|---------|----------|---------|
| **运行环境** | `ENVIRONMENT=development`, `DEBUG=true` | 必须改为 `production` + `false` | 高 |
| **JWT Secret Key** | 可能是默认值 `dev-xxx` | 必须设置强随机秘钥 | 高 |
| **管理员密码** | `FIRST_ADMIN_PASSWORD` 可能是默认 | 必须修改 | 高 |
| **MinIO 密钥** | `hermes-minio-secret` (默认) | 必须修改 | 高 |
| **HTTPS 证书** | 自签名证书 | 确认过期时间，是否需要 Let's Encrypt | 中 |
| **数据库备份** | 只有 1 次手动备份（6/23） | 必须建立自动化备份（每日/每周） | 高 |
| **公网访问** | 当前只支持内网 | 是否需要 VPN/公网映射/零信任 | 中 |
| **登录方式** | 本地账号 + LDAP + 企业微信 | 确认主要登录方式，测试流程 | 中 |

## 四、压力测试方案

### 4.1 当前性能基线

| 指标 | 当前值 | 备注 |
|------|---------|------|
| Agent 并发上限 | 10 个会话同时推理 | 刚从 5 调整 |
| API 连接池 | 30 个 | pool_size=10 + max_overflow=20 |
| 单用户限流 | 30 条/分钟 | 可通过环境变量覆盖 |
| 推理超时 | 900 秒 | 超时后自动重试 3 次 |
| 服务器 CPU | 32 核 | 远程 API 推理，本地几乎不占 CPU |
| 服务器内存 | 30GB，可用 17GB | 可支撑 10 个并发 + 大量余量 |

### 4.2 建议的压力测试场景

| 场景 | 目标 | 测试方法 |
|------|------|----------|
| 场景 A：5 人同时发送简单问题 | 响应 < 5 秒 | 手动或脚本模拟 |
| 场景 B：10 人同时发送 | 全部响应，无丢失 | 使用 k6 或 Locust |
| 场景 C：1 人持续发送 50 条 | 触发限流，429 错误 | 测试限流策略是否正常 |
| 场景 D：长文本输入 | 消息不丢失 | 测试大于 2000 字的输入 |
| 场景 E：重启恢复 | 1 分钟内全部恢复 | 模拟重启后所有服务自启 |

### 4.3 压力测试工具建议

```bash
# 快速手动测试 API 健康
watch -n 1 'curl -s https://hermes.infiled.com/api/v1/healthz | head -c 100'

# 快速测试登录
```

## 五、风险矩阵

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 运行环境未切换到 production | 高 | 高 | 发布前必须修改 `.env` |
| 密钥/密码使用默认值 | 高 | 高 | 全部更换为强密码 |
| 没有自动化数据库备份 | 中 | 高 | 配置 pg_dump 定时任务 |
| 服务器意外重启 | 低 | 中 | 验证 systemd enable 状态 |
| 红色累计流量爆棚 | 低 | 中 | 监控 Redis 和数据库连接池 |
| 证书过期 | 低 | 中 | 记录过期时间，设置提醒 |
| 用户不熟悉产品 | 中 | 低 | 准备简短使用指南 |

## 七、附录：快速操作参考

### 查看服务状态
```bash
systemctl --user status hermes-api hermes-web hermes-agent-runner
ss -tlnp | grep -E "8001|5173|443"
```

### 查看实时日志
```bash
journalctl --user -u hermes-api -f
journalctl --user -u hermes-agent-runner -f
```

### 重启所服务（用户手动执行）
```bash
systemctl --user restart hermes-api hermes-web hermes-agent-runner
caddy reload --config /home/hermes/hermes-infi-webui/Caddyfile
```

### 数据库手动备份
```bash
pg_dump -U hermes -d hermes > hermes_backup_$(date +%Y%m%d).sql
```

### 修改环境变量（发布前必做）
```bash
# 编辑 /home/hermes/hermes-infi-webui/backend/.env
ENVIRONMENT=production
DEBUG=false
SECRET_KEY=<强随机字符串>
FIRST_ADMIN_PASSWORD=<强密码>
MINIO_SECRET_KEY=<强随机字符串>
```
