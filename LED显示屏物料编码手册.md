# LED显示屏物料编码手册

> 适用系统：金蝶云星空 / 金蝶K/3 / 用友U8  
> 编码总长度：≤30位（兼容U8限制）  
> 版本：V1.1

---

## 一、编码总则

### 1.1 字符集
- **仅允许**：大写字母 A-Z、数字 0-9、连字符 `-`
- **禁用字符**：`I`、`O`、`L`、`Z`（易与1/0混淆）
- **全大写**：不得出现小写字母

### 1.2 结构范式

**原材料/零部件**：
```
[大类码 2位] + [中类码 2位] + [分隔符 -] + [规格码 4-6位] + [版本/流水 2-3位]
```

**成品/半成品**：
```
[大类码 2位] + [产品系列 2位] + [分隔符 -] + [像素间距 2位] + [规格码 4-6位] + [版本码 2位]
```

| 段 | 位数 | 说明 |
|----|------|------|
| 大类码 | 2位字母 | 物料所属一级类别 |
| 中类码/系列 | 2位字母 | 二级类别或产品系列 |
| 分隔符 | 1位 | `-`，增强可读性 |
| 规格码 | 4-6位 | 规格/型号/尺寸特征 |
| 版本码 | 2-3位 | 版本或流水号 |

**典型长度**：11-16位，预留余量给U8的30位上限。

### 1.3 编码唯一性原则
- 一物一码，不因供应商不同而变更编码
- 规格、封装、色温等关键属性变化时，必须赋新码
- 颜色/色温差异必须体现在规格码或版本码中
- 替代料不走编码层面，通过BOM替代料功能维护

## 三、成品编码规则（FP）

### 3.1 成品大类结构

```
FP-[产品系列2位]-[像素间距2位]-[箱体尺寸3位]-[安装方式1位]-[版本2位]
```

| 位段 | 说明 | 位数 | 取值 |
|------|------|------|------|
| 产品系列 | 产品定位/应用系列 | 2位 | 见3.2节 |
| 像素间距 | 点间距 mm×10 | 2位 | 25=P2.5 39=P3.9 48=P4.8 |
| 箱体尺寸 | 箱体尺寸 mm/10 | 3位 | 050=500mm 100=1000mm |
| 安装方式 | 安装/应用方式 | 1位 | 见3.3节 |
| 版本码 | 版本/配置 | 2位 | 01-99 / A0-Z9 |

**总长度**：13位

### 3.2 产品系列码表

| **SM** | Small Pixel | 小间距系列 | 会议/监控/指挥室 |
| **CR** | Creative | 创意屏系列 | 异形/定制造型 |

### 3.5 成品版本变更规则

| 版本类型 | 触发条件 | 示例 |
|----------|----------|------|
| 小版本+1 | 非关键BOM变更、供应商切换、丝印调整 | 01→02 |
| 中版本A/B | 关键器件变更（IC/灯珠批次）、结构微调 | 01→A0 |
| 大版本V1/V2 | 平台换代、重大结构变更、间距不变整机重设计 | 01→V1 |
| 配置后缀 | 同一版本不同配置（亮度/刷新率） | `FP-RE-39-050-F-01-H` = 高亮版 |

**配置后缀表**（可选，加在版本后，用 `-` 分隔）：

| 后缀 | 含义 |
|------|------|
| H | High Brightness 高亮版 |
| S | Standard 标准版 |
| E | Economy 经济版 |
| U | Ultra 旗舰版 |
| C | Custom 客制版 |

## 四、半成品编码规则（SP）

### 4.1 半成品大类结构

```
SP-[半成品类型2位]-[像素间距2位]-[尺寸/规格4位]-[版本2位]
```

| 位段 | 说明 | 位数 | 取值 |
|------|------|------|------|
| 半成品类型 | 半成品形态 | 2位 | 见4.2节 |
| 像素间距 | 点间距 mm×10 | 2位 | 25=P2.5 39=P3.9 |
| 尺寸/规格 | 尺寸或特征码 | 4位 | 见各类型说明 |
| 版本码 | 版本/配置 | 2位 | 01-99 |

**总长度**：12位

### 4.2 半成品类型码表

| 代码 | 类型 | 说明 |
|------|------|------|
| **MO** | **Module** | **LED模组（带面罩成品模组）** |
| **PC** | **PCBA** | **贴装完成的PCBA板** |
| **LP** | **Light Plate** | **灯板（已贴灯珠未贴IC）** |
| **BP** | **Back Plate** | **背板/后盖组件** |
| **PS** | **Power Set** | **电源组件（含电源+线材）** |
| **CS** | **Control Set** | **控制组件（含接收卡+线材）** |
| **FS** | **Frame Set** | **箱体框架组件** |
| **AS** | **Assembly Set** | **半装组件（模组+箱体未接线）** |

### 4.3 各类型编码细则

```
SP-MO-[像素间距2位]-[模组尺寸4位]-[版本2位]
```

| 位段 | 说明 | 取值 |
|------|------|------|
| 模组尺寸 | 长×宽 mm/10 | 2525=250×250mm 5025=500×250mm |

- `SP-MO-25-5025-01` = P2.5 500×250mm 模组

```
SP-LP-[像素间距2位]-[板尺寸4位]-[灯珠色1位]-[版本2位]
```

## 六、零部件规格码编制规则

### 6.1 LED灯珠（DR-SM/DR-CB）

格式：`DR-SM-[尺寸4位]-[颜色1位]-[芯片厂1位]-[流水2位]`

| 位段 | 说明 | 取值 |
|------|------|------|
| 尺寸 | 封装尺寸 | 1515/2121/2727/3535/1921 |
| 颜色 | 发光颜色 | R=红 G=绿 B=蓝 W=白 Y=黄 A=琥珀 |
| 芯片厂 | 芯片供应商 | G=国星 J=晶台 T=三安 N=诺瓦 |

**示例**：
- `DR-SM-2727-R-G-01` = 国星2727红色贴片灯珠，版本01
- `DR-SM-1515-B-J-01` = 晶台1515蓝色贴片灯珠
- `DR-CB-1616-W-G-01` = 国星1616白色COB灯珠

### 6.2 驱动IC（IC-DR）

格式：`IC-DR-[型号5位]-[批次2位]`

| 位段 | 说明 |
|------|------|
| 型号 | 取原型号数字部分，不足补零 |
| 批次 | 00=通用，01=首批，02=替代批次 |

**示例**：
- `IC-DR-2053-00` = ICN2053（取后四位2053）
- `IC-DR-6126-00` = FM6126（取后四位6126）
- `IC-DR-5153-01` = MBI5153首批

> **注意**：IC型号不规则时，建议维护一张映射表，编码仅取映射序号。例如：`IC-DR-0012-00` 映射到 ICN2053。

### 6.3 PCB板（PC-HD/PC-AL）

| 封装 | 封装代码 | 04=0402 06=0603 08=0805 |

**示例**：
- `EL-RS-1001-1-06-01` = 1KΩ ±1% 0603贴片电阻
- `EL-CA-104K-5-08-01` = 100nF ±10% 0805贴片电容

## 七、套件/组合物料编码（KT）

### 7.1 套件编码结构

```
KT-[用途2位]-[组成数2位]-[流水3位]
```

| 位段 | 说明 | 取值 |
|------|------|------|
| 用途 | 套件用途 | MD=模组套 BX=箱体套 PJ=项目套 |
| 组成数 | 组成物料数量 | 04=4件 06=6件 |

**示例**：
- `KT-MD-04-001` = 模组套件（含PCB+灯珠+IC+面罩）
- `KT-BX-08-001` = 箱体套件（含箱体+模组+电源+控制卡+线材+面罩+背板+配件）
- `KT-PJ-12-001` = 项目套件（整屏装配包）

### 7.2 套件BOM引用规则

套件在系统中以虚拟件形式存在，BOM展开到下层实物物料。编码层面不做嵌套套件。

## 八、版本码规则

### 8.1 通用版本规则

| 版本码 | 含义 |
|--------|------|
| 01-99 | 初始版本，顺序递增 |
| A0-Z9 | 大改款（结构/电气变更） |
| X0 | 停产/淘汰 |

### 8.2 变更触发条件

| 变更类型 | 触发条件 | 版本变化 |
|----------|----------|----------|
| 小版本+1 | 供应商变更、丝印调整、非关键参数微调 | 01→02 |
| 中版本A0 | 关键器件变更、结构微调、电气参数调整 | 01→A0 |
| 大版本V1 | 平台换代、整机重设计、间距不变架构变更 | 01→V1 |
| 停产码X0 | 物料彻底停产，库存清零后冻结 | 任何→X0 |

### 8.3 版本跨类继承

当成品版本升级由半成品变更驱动时：
- 半成品版本独立升级
- 成品BOM同步更新半成品引用
- 成品版本按需升级（不一定每次半成品变更都升成品版本）

## 九、客制化/项目专用规则

### 9.1 项目专用物料

项目专用物料在标准编码后加项目后缀：

```
[标准编码]-[项目码3位]
```

**示例**：
- `FP-RE-39-050-F-01-P01` = P01项目专用租赁屏
- `SP-MO-39-2525-01-P01` = P01项目专用模组
- `DR-SM-2727-R-G-01-P01` = P01项目专用红色灯珠

### 9.2 客户定制版本

客户定制在版本码后用 `-C` 标识，后续跟客户代码：

```
[标准编码]-C[客户码2位]
```

**示例**：
- `FP-RE-39-050-F-01-C01` = 01号客户定制租赁屏

## 十、替代料管理规则

替代关系不在编码中体现，通过金蝶BOM替代料功能维护。编码层面保持独立。

替代料维护原则：
- A料为主料，B料为替代料
- 替代关系需经工程部审批
- 替代比例、优先级在BOM中设定

## 十一、金蝶系统配置建议

### 11.1 物料属性映射

| 属性 | 金蝶字段 | 填写规则 |
|------|----------|----------|
| 物料编码 | FMaterialID | 按本手册编制 |
| 物料名称 | FName | 中文名称，与编码对应 |
| 规格型号 | FSpecification | 完整规格参数 |
| 物料属性 | FErpClsID | 外购/自制/委外 |
| 计量单位 | FUnitID | 个/片/米/套/千克/平方米 |
| 默认仓库 | FStockID | 按类别分配 |

### 11.2 辅助属性建议

启用以下辅助属性，避免过度细分编码：
- 颜色（红/绿/蓝/白/RGB）
- 色温（6500K/3200K等）
- 亮度等级（A/B/C级）
- 刷新率（1920Hz/3840Hz等）
- 供应商
- 项目归属

### 11.3 条码规则

条码内容：`{物料编码}-{批次号}-{数量}`

建议采用 Code128 或 QR Code，确保字母数字均可识别。
